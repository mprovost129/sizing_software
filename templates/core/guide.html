Quick Start Guide

How to size a member, read the results, and save your work

Overview

FrameCalc sizes residential framing members using code calculations, then tells you whether a given member passes and by how much. It is modeled on the way tools like BC CALC work: set your span, apply loads, pick a member, and analyze.

The current release checks a sawn-lumber material library (currently SPF, Douglas Fir-Larch, and Hem-Fir grades over 2x4 through 2x12 sizes) against NDS 2018 and IRC R301.7, with single-span support fully in place and a first-pass continuous workflow for 2-span through 10-span members, including end overhangs, now available. Results are preliminary sizing estimates, not stamped engineering.

The design workflow

A new design has three input tabs and two ways to analyze.

1
Span   Set one span by default, then continue through the continuous-span rows as needed up to 10 spans total.
2
Loads   Enter standard load components as either area loads (psf) or line loads (plf), plus any point loads.
3
Settings   Choose the member type, deflection criteria, and the nominal size to check.
→
Then either Run Analysis to check the selected size, or All Sizes to scan every section and get the most economical one that passes.

1 · Span tab

Define the geometry of the member and its supports.

•
Span 1 / Span 2 / Span 3 plus the additional rows define one clear span or a first-pass continuous member with up to ten spans.
•
Left / right overhangs can be combined with that continuous layout, and the member preview plus cantilever tip checks will follow the full beam geometry.
•
Support types choose how each support is carried: wall/plate, column/post, or hanger. This drives the support symbol drawn in the loading diagram.
•
Bearing lengths at each support feed the bearing (Fc-perp) checks and the bearing area factor Cb.

2 · Loads tab

Enter at least one load. Uniform loads are the tributary load on this member.

•
Uniform load basis lets you work in area load (psf) or line load (plf). psf entries convert to plf using the on-center spacing.
•
D / L / S / Lr / W are available as full-length uniform components. Each carries its own load-duration factor (CD): dead 0.9, live 1.0, snow 1.15, roof live 1.25, wind 1.6.
•
Point loads take a magnitude (lb), a location from the left support (ft), and a load type. Add up to three.
The engine evaluates each relevant load combination separately and keeps the governing one for each check, the same way you would verify by hand.

3 · Settings tab

Pick the section and how it is used.

•
Object toolbar picks the member workflow first, more like BC Calc's joist / beam / rafter object buttons.
•
Material selects the sawn-lumber species/grade library entry used for the checks, separate from the nominal size buttons.
•
Member type (floor joist, ceiling joist, rafter, beam/header) seeds the default deflection limits per IRC Table R301.7.
•
Back-span and cantilever deflection criteria are editable, matching the BC Calc-style Settings workflow.
•
Performance target tightens the recommended deflection criteria above bare IRC minimums when you want a stiffer feel or harder-finish focus.
•
Subfloor / floor feel further tightens floor-joist serviceability targets only; it does not assume composite action or add structural capacity.
•
Member size is a row of buttons (2x4 to 2x12). Click to select; double-click to select and immediately run.
•
Repetitive member applies Cr = 1.15 to bending when members are ≤ 24 in o.c., 3 or more, joined by sheathing.
•
Design name is optional and only used when you save.
•
Project is optional when saving. You can pick an existing project or type a new one inline to start grouping related designs.

Running an analysis

Two buttons, two behaviors.

•
Run Analysis checks the currently selected size and fills the results panel. Double-clicking a size button does the same for that size.
•
All Sizes runs every section and shows a scan table, highlighting the most economical passing member. This is currently single-span only.

Reading the results

The results panel leads with a pass/fail verdict, then the detail.

•
A green PASS or red FAIL banner names the section and the governing check. Failing checks are shown in red.
•
Stat pills show the section properties (A, S, I) and the size and repetitive factors used.
•
The loading diagram draws the beam, supports, distributed load, and reactions to scale.
•
The load combinations table lists reactions, max shear, and max moment per combo.
•
The design checks table gives demand, capacity, utilization ratio, and governing combo for each check.

The six checks:

Bending
fb = M/S vs F'b = Fb × CF × Cr × CD
Shear
fv = 1.5V/A vs F'v = Fv × CD
Bearing L / R
R / (b × Lb) vs Fc-perp × Cb, at each support
Deflection (live)
transient-load deflection vs the live limit (e.g. L/360)
Deflection (total)
total-load deflection vs the total limit (L/240)

Scanning all sizes

The All Sizes view is how you find the smallest member that works.

•
Every section is listed with each check ratio color-coded green, amber (tight), or red (fails).
•
The most economical passing size is flagged Best and highlighted.
•
Double-click any passing row to load that size back into the designer.

Saving & designs

Keep a record of a member you have sized.

•
Give the design a name in Settings, optionally assign or create a project there, then click the save (disk) button.
•
Designs in the top nav groups everything you have saved by project, with each design still showing its pass/fail status and governing ratio.
•
Open a saved design to see the full results, export it to CSV, or delete it. Saved designs recompute from stored inputs, so results stay current if the engine is refined.
